# 通话 UI 流程设计

## 页面与组件清单

### 新增组件

| 组件 | 位置 | 职责 |
|------|------|------|
| `CallProvider` | 全局 Context | 管理通话状态机，监听来电 |
| `IncomingCallModal` | 全局覆盖层 | 来电弹层 |
| `ActiveCallScreen` | 全屏页面 | 通话中主界面 |
| `CallStatusBanner` | ChatDetail 顶部 | 通话状态提示 |
| `CallMessageRenderer` | 消息列表中 | 渲染通话记录消息 |

### 修改组件

| 组件 | 变更 |
|------|------|
| `ChatDetail` | 顶部添加语音/视频通话按钮（仅单聊） |
| `App.tsx` | 包裹 CallProvider |

## UI 流程

### 流程 1：发起通话

```
ChatDetail
  └─ 用户点击 📞 或 📹
  └─ 权限检查（麦克风/摄像头）
      ├─ 拒绝 → Toast提示 "请允许访问麦克风"
      └─ 允许 → 创建 call_session
          └─ 进入 ActiveCallScreen（dialing 状态）
              ├─ 显示对方头像、昵称
              ├─ 显示 "正在呼叫..."
              ├─ 取消按钮
              └─ 等待对方响应
```

### 流程 2：收到来电

```
任意页面
  └─ Realtime 收到 call_sessions 变更（status=ringing, callee=当前用户）
  └─ 弹出 IncomingCallModal
      ├─ 显示来电人头像、昵称
      ├─ 显示通话类型（语音/视频）
      ├─ 接听按钮（绿色）
      ├─ 拒绝按钮（红色）
      └─ 用户操作：
          ├─ 接听 → 请求权限 → 加入房间 → ActiveCallScreen
          └─ 拒绝 → 更新 status=rejected → 关闭弹层
```

### 流程 3：通话中

```
ActiveCallScreen
  ├─ 视频通话：
  │   ├─ 对方画面（全屏）
  │   ├─ 本地预览（右上角小窗，可拖动）
  │   └─ 底部控制栏
  │
  ├─ 语音通话：
  │   ├─ 对方头像（居中，大尺寸）
  │   ├─ 对方昵称
  │   ├─ 通话时长计时器
  │   └─ 底部控制栏
  │
  └─ 控制栏按钮：
      ├─ 静音/取消静音（麦克风图标）
      ├─ 开关摄像头（仅视频通话）
      ├─ 切换前后摄像头（仅视频通话）
      ├─ 扬声器切换
      └─ 挂断（红色圆形按钮）
```

### 流程 4：通话结束

```
ActiveCallScreen
  └─ 挂断 / 对方挂断 / 网络断开
  └─ 显示结束状态（1.5秒）
      ├─ "通话已结束 02:14"
      ├─ "对方已挂断"
      └─ "网络连接断开"
  └─ 自动返回聊天页
  └─ 聊天记录中插入通话消息
```

## 视觉规范

### 配色
- 接听按钮：绿色 `#22C55E`
- 挂断/拒绝按钮：红色 `#EF4444`
- 通话背景：深色渐变 `#1a1a2e` → `#16213e`
- 控制按钮：半透明白色 `rgba(255,255,255,0.2)`
- 激活状态：品牌橙 `#F97316`

### 布局
- 全屏通话页，隐藏底部导航
- 安全区域适配（safe-area-inset）
- 控制栏距底部 40px
- 本地预览窗口：120x160px，圆角 12px
