## 概述

阿基米●聊 (Archimi Chat) 包含一对一语音和视频通话功能，使用 100ms 作为音视频基础设施。

## 测试步骤

### 测试 1：语音通话（需要两台设备）

1. 设备 1 登录账号 A，设备 2 登录账号 B
2. 账号 A 进入与账号 B 的聊天页面
3. 点击顶部 📞（电话图标）
4. 系统会请求麦克风权限 → 允许
5. 屏幕显示"正在呼叫..."
6. 设备 2 收到来电弹层，显示来电人信息
7. 点击"接听"
8. 双方进入通话界面
9. 测试静音/取消静音按钮
10. 点击挂断结束通话
11. 聊天记录中出现通话消息，显示通话时长

### 测试 2：视频通话（需要两台设备）

1. 同上步骤 1-2
2. 点击顶部 📹（视频图标）
3. 系统请求麦克风和摄像头权限 → 允许
4. 对方接听后，双方可看到对方画面
5. 测试切换前后摄像头按钮
6. 测试关闭/开启摄像头按钮
7. 挂断结束通话

### 测试 3：拒绝通话

1. 账号 A 发起语音通话
2. 账号 B 点击"拒绝"
3. 账号 A 看到"对方已拒绝"提示
4. 聊天记录中出现"[语音通话] 已拒绝"

### 测试 4：未接听

1. 账号 A 发起语音通话
2. 账号 B 不操作，等待 60 秒
3. 通话自动结束，显示"未接听"
4. 聊天记录中出现"[语音通话] 未接听"

### 测试 5：权限拒绝处理

1. 在系统设置中关闭 App 的麦克风权限
2. 尝试发起语音通话
3. 应显示友好提示"需要麦克风权限"，而非崩溃
4. 提示中有"去设置"按钮

## 单设备验证（如审核员只有一台设备）

如果只有一台设备，可以验证：

1. **发起通话 UI**：登录账号 A，进入与 B 的聊天，点击通话按钮，观察"正在呼叫"界面和取消按钮
2. **权限处理**：拒绝权限后的提示是否正确
3. **通话记录**：聊天中的通话消息显示
4. **按钮状态**：通话界面的控制按钮是否响应

## 权限使用说明

| 权限 | 使用场景 | 用户触发方式 |
|------|---------|-------------|
| 麦克风 | 语音/视频通话 | 用户主动点击通话按钮 |
| 摄像头 | 视频通话 | 用户主动点击视频通话按钮 |

权限仅在用户主动发起或接听通话时请求，App 不会在启动时或其他场景请求这些权限。

- 本应用不包含直播/聊天室功能
- 通话功能仅限已添加好友的用户之间使用
